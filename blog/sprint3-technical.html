HTML, CSS and the DOM

An analogy for HTML and CSS

Hypertext Markup Language (HTML) is used to create the contents of the page, and the Cascading Styling Sheet (CSS) is used to display things, well, "nicely".

To put this into an easy analogy, we can think of HTML and CSS like a book. HTML is all the content, but the book would look rather silly if the content wasn't arranged into a "book".

This is where CSS comes into play. CSS is responsible for arranging the contents and decide how the book is going to look for the person reading it!


Control Flow and Loops

To put it easily, control flow is the order execution of a script.

Normally, the script is read by the computer from top to bottom. However, if we introduce control flow, it creates conditional statements to control how the code runs!

An example of a control flow is the 'if' statement, where a condition requires to be met for the code to execute. For instance:

if (cat == hungry) {
callButler();
} else {
sleep();
}

This shows that if the cat is hungry, it will call its Butler, and in any other case, it would sleep.


Now, looking at loops, it sounds pretty self explanatory right? It's when an action is repeated until a condition is met.

Let's take drawing cards as an example:

do {
drawCard();
} while (hand < 5)

In the above example, the code will repeat itself until the number of cards in his hand is LESS than 5 (so 4 cards). You can see here how control flow can help execute code under different conditions.


What the DOM?!

"DOM" isn't a word you'd usually say or hear in your everyday life. DOM stands for Document Object Model and it treats what we see on the pages (HTML) as a tree structure.

It helps us to interact with various elements on the page you see, and we can even edit the page using the DOM and allows us to be much more flexible with JavaScript as well!


Accessing arrays and objects

To start this topic, it would be helpful if I explain what an array and object is.

Easily put, an array is a list of objects, such as a shopping list, and an object is something with properties and values for those properties. I'm sure the following information will make this a little easier to understand.

var array = ["cat", "dog", "cow", "chicken"]

var object = {
name: "Objecto",
legs: 4,
ethnicity: "Martian",
likes: ["food", "cats", "garlic"]
}

In order to call upon arrays, we use indexing via the [] brackets, and the first object in the array is [0], which would be "cat", and [3] would be "chicken" which is in the 4th position.

For objects, there are 2 ways to access them, using . or []. For example, to access the name of the object, we could write object.name or object["name"], which would show its name.

A large portion of what we do with coding is called Object Orientated Programing (OOP), and involves the use of objects and acessing them.


Functions: What are they and how are they useful?

A function is a fundamental part of coding, and it is a JavaScript procedure. It is a set of statements that performs a task (function) depending on how we code it.

A very basic function would be something that does addition, subtraction, multiplication etc., and here is an example.

function addition(a, b) {
return = a + b;
}
console.log(addition(3, 5))

and this would return 8 as the answer.

We can now call on the addition function anytime with different arguements (i.e. addition(10, 5) to get 15), as we have created it as a function.
